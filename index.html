Client approached me over application format: standard expectation at the time was a 'smart' pdf with backup paper materials provided. This traditional application process required the HR and MGMT group to keep track of multiple formats per each applicant. Materials were cross checked in case of PDF issues. TropeCollaborative.com

Third generation representation of client group. Site base changed to WIX as expeditionary test of database and data handling integrated CMS. Some tricks and layout hacks inplemented to counter limited layout and formatting options. Results obtained in spite of lack of CSS editing. linkSite
more

The media heavy sections for the site were built using database objects rather then specifically build the links. Wix editing system excells at dynamic data but lacks any specialized CSS or layout tools. Left oriented layout mathmatics were utilized to keep pull quotes in their specified order without covering up actual content. Specially ordered placeholders were populated for the area the quotes are placed. According to WIX documentation, many of the results we produced were not possible.

Kallish Holiday Site (yearly)

Yearly holiday site for Clients' family. Images and layout iterated each year. Grid layout with adjustment and detail views. Detail view tagged with format so that different aspect ratios are aligned to different grid layouts. Features built on top of core Masonry plugin layout. linkSite
more

Images are folder hosted. Each image div-group has attributes attached in order to process animations and layout changes without having to produce a more complex logic system. Simpler and faster returns meant less development cost. Layout pulls an attribute from the image hosting div to adjust Masonry layout. Masonry plugin customized to use a mathematically set grid, so that varying image aspect ratios are sill within grid alignment.
